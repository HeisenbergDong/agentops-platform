# 09 GitHub Rules

- Submit GitHub only after verified full Trae trace exists.
- Ensure .gitignore excludes node_modules, dist, build, target, .venv, __pycache__, and generated caches.
- Commit messages should include project, round, and short task purpose.
- Push failures abort Feishu business write.
- Normalize GitHub URL for Feishu as `https://github.com/owner/repo.git`.
- Convert SSH remotes like `git@github.com:owner/repo.git` to HTTPS clone URL.
- Network, credential, permission, or remote errors must create automation errors.
